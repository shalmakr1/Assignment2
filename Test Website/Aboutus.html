The web browser serves as a doorway to the Internet for much of a typical user's online activity. Common privacy concerns come up when simply browsing the web. Browsers provide common tools enabling users to interact with remote servers in a secure fashion, 1) Encrypted sessions (SSL) 2) Signed Certificates.

One need a trustworthy access to internet. Some misleading websites try to cause harm to user in term of security by following ways: 1- Malicious sites trying to use a forged identity that are often related to phishing attacks 2- More elaborate impersonation attacks are also possible using JavaScript 3- Again, all technically legitimate JavaScript commands, used with the intention of misleading the user. Then to increase the privacy of users, World Wide Web Consortium (W3C) developed a policy with the name of Privacy Preferences Project (P3P). It offers an easy way for web sites to communicate and enables the development of tools. It checks the website in following steps: First Step: name and contact information for site Second Step: kind of access provided Third Step: mechanisms for resolving privacy disputes Fourth Step: kinds of data collected.
Infact, Browsers should trade off security for compatibility.
